Working loop with coding agents

Treat the agent as a collaborator inside a system, not as a slot machine for code. The loop is small on purpose: orient, constrain, execute, verify, preserve.

One practical loop

  1. Orient: read the repo and name the next safe move.
  2. Constrain: write goal, context, constraints, and done criteria.
  3. Execute: give the agent one bounded slice.
  4. Verify: run the strongest cheap check you have.
  5. Preserve: write the decision into the repo before it evaporates.

Signals that the loop is healthy

  • the next person can find the entry point quickly
  • the repo says what was verified and what was not
  • constraints live in files, not just in chat
  • the agent is solving one problem at a time

Signals that the loop is drifting

  • you keep re-explaining the same rule in prompts
  • the diff looks busy but the trust boundary is weak
  • tests were generated by the same pass that wrote the code
  • no one can tell what the next safe move is

What to keep in the repo

  • AGENTS.md
  • build and test commands
  • one review or handoff checklist
  • a short runbook when the same friction repeats
