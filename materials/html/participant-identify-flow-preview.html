Participant identify flow preview

The participant flow is intentionally narrow: one shared room key, one name choice, one personal password. It gives the room a clean entry path without turning the participant surface into an admin tool.

Flow

  1. Enter the shared event code on the public homepage.
  2. Pick your name from the room roster.
  3. Set a password on first entry, or enter it on return.
  4. Land in the participant room with the live workshop context.

Why this works

The shared code unlocks the room context, but each participant still has their own ongoing identity. The facilitator does not need to hand out individual credentials in advance.

What it avoids

No facilitator password leakage. No free-text identity guessing. No need to re-enter the shared code for every casual return.

Walk-ins

If walk-ins are enabled, a participant whose name is not in the prepared roster can still enter by creating a room-safe display identity through the supported path.

Password recovery

Password resets stay on the facilitator side. The participant surface should never collect or relay someone else’s password.
