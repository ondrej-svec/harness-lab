## The three agenda.json span overrides (a real limitation honestly stated)

The walker tags spans structurally: any string value under a `phases[].cs.*` key path is tagged `cs`. This is correct for 99% of agenda content. But three scenes contain **English strings inside Czech scene branches** — commitment examples quoted in English for teaching purposes, and one English facilitator note. The walker cannot see sub-string language changes.

The fix for these three is a direct lockfile override: change `language: "cs"` → `language: "unknown"` on the affected spans. Each override is documented in the span's `note` field.

| Span | pathHint | Content |
|---|---|---|
| `5204:29` | `phases.scenes.cs.blocks.items.body` | `"No 'I will try…....'"` — English vague-commitment counter-example |
| `5230:25` | `phases.scenes.cs.blocks.body` | `"Not: 'I'll work better with agents.' Yes: 'The next tim I start...'"` — English commitment examples inside a Czech callout body |
| `5290:15` | `phases.scenes.cs.facilitatorNotes` | `"30-60 seconds of screen time..."` — English facilitator note |

A future `StructuralSegmenter` backend in the toolkit could detect these via diacritic density, stopword frequency, or an LLM-assisted pre-pass that uses the agent-side v2 prompt's quote-handling logic. For now, three manual overrides are a cheap and reviewable substitute.

## CI integration

`dashboard/package.json` gains two scripts:

```json
"verify:content": "... && bun ../scripts/content/verify-copy-editor.ts",
"verify:copy-editor": "bun ../scripts/content/verify-copy-editor.ts"
```

The `verify-copy-editor.ts` wrapper discovers the copy-audit script by trying three locations in order:

1. `$HEART_OF_GOLD_TOOLKIT/plugins/marvin/skills/copy-editor/scripts/copy-audit.ts` — explicit env var for CI.
2. `../heart-of-gold-toolkit/plugins/marvin/skills/copy-editor/scripts/copy-audit.ts` — sibling repo (developer layout).
3. `~/.claude/plugins/marketplaces/heart-of-gold-toolkit/...` — Claude Code plugin marketplace cache.

It invokes `copy-audit --require-reviewed` and propagates the exit code. A failing gate reports the offending files and rules inline.

**Current state:** `npm run verify:copy-editor` exits 0 (no findings, all entries reviewed). `npm run verify:content` runs the full chain — brief generator verify, views generator verify, tier2 sync check, and copy-editor — all green.

## How to keep it green going forward

1. **When content files change** — `--list-unsegmented` surfaces the stale files on next audit. The host agent re-segments via `copy-audit segment <path>` → `lockfile add --spans -`.
2. **When pure-Czech files accumulate real typography errors** — `copy-audit --fix` handles R1 (nbsp after preposition), R1b (nbsp after conjunction), and R3 (ellipsis) mechanically.
3. **When a file mixes languages** — split the lockfile span by hand or (eventually) let the agent-side segmenter re-run with the v2 prompt's inline-code / quote-handling rules.
4. **Before merge** — `npm run verify:content` enforces the gate. A failing run lists the offending files.

## Deferred / open items

- **`StructuralSegmenter` backend in the toolkit** — the `/tmp/segment-agenda.ts` walker used in this rollout is the rough shape of the eventual backend. Upstreaming it would close Phase 3 of the locale-segmentation plan and eliminate the need for one-off helper scripts.
- **Sub-string English-in-Czech detection** — the three agenda.json overrides are a manual workaround for a real architectural limitation. The long-term fix is either a diacritic-density heuristic in the structural backend, or an agent-side re-segmentation pass that uses the v2 prompt's quote-handling rules.
- **Review workflow for lockfile PR diffs** — when the content plan's Phase B15 (Czech regeneration) eventually lands real translations, every affected file goes stale. The rollout pattern from this session is the playbook: `--list-unsegmented` → agent re-segments → `lockfile add` → `mark-reviewed` → `verify:content`.
