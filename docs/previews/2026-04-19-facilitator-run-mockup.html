Design-system note

Reused rules vs. doc follow-through

Reused unchanged: calm-before-clever styling, iPad-first composition, explicit action hierarchy, mobile-safe summary cards, and the agenda spine as navigation.
Docs to update: facilitator product contract, control-room default surface, and surface model text that still treats agenda/scene authoring as dashboard work.
Proof-slice test implication: the dominant Playwright path becomes “land on Run, confirm current/next/safe-action, open presenter, move live.”
