# Autonomous Planning Examples

These examples come from real Harness Lab work.

The point is not to shame earlier docs. The point is to make the planning signal explicit enough that future autonomous work does not have to rediscover the same gaps.

## Example 1: Strong Product Thinking, Weak Subjective Gate

Source:

- [`brainstorms/2026-04-08-facilitator-presenter-rich-content-redesign-brainstorm.md`](./brainstorms/2026-04-08-facilitator-presenter-rich-content-redesign-brainstorm.md)

What it did well:

- named the real problem instead of asking for generic polish
- explained what the current model could not express
- defined a strong structural direction and clear decisions

What it still lacked for autonomous subjective execution:

- no anti-references for what the presenter must not become
- no explicit rejection criteria for projection-safe visual drift
- no required preview artifact before implementation
- no proof-slice gate that separated "prove one pack" from "roll the new pattern everywhere"

Lesson:

Good product reasoning is necessary but not sufficient for autonomous design-heavy work.

## Example 2: Better Editorial Feedforward, Still Missing Preview-First Discipline

Source:

- [`brainstorms/2026-04-07-public-homepage-copy-brainstorm.md`](./brainstorms/2026-04-07-public-homepage-copy-brainstorm.md)

What it did well:

- explored multiple approaches and tradeoffs
- named audience-facing anti-patterns
- included assumption audit and concrete draft directions
- treated natural Czech as a real constraint

What it still lacked:

- no explicit anti-reference set at plan time
- no proof-slice rollout rule
- no visual preview artifact even though copy and layout interact on the homepage

Lesson:

Copy-heavy work is often also layout-heavy. If the message hierarchy depends on the screen, prose-only planning is still incomplete.

## Example 3: Strong Subjective Quality Bar, Ready For Hardening

Source:

- [`plans/2026-04-09-feat-workshop-scene-editorial-rollout-and-copy-excellence-plan.md`](./plans/2026-04-09-feat-workshop-scene-editorial-rollout-and-copy-excellence-plan.md)

What it did well:

- set an explicit editorial quality standard
- named the weak scenes instead of hiding behind broad goals
- defined source-first rewriting and verification discipline
- treated Czech quality and facilitation usefulness as load-bearing

What the hardened standard adds:

- explicit anti-references
- explicit rejection criteria at the plan gate
- preview-first requirement for design-heavy or room-facing changes
- helper-role review before broad rollout

Lesson:

This is close to the right shape. The hardening work is about making the proof gate and preview gate explicit.

## What A Hardened Subjective-Work Input Looks Like

Before autonomous `work`, the plan or brainstorm should already make these items obvious:

- the target outcome
- the anti-goals
- the references
- the anti-references
- the tone or taste rules
- the representative proof slice
- the rollout rule
- the rejection criteria
- the required preview artifact

If one of those is missing, expect drift.
