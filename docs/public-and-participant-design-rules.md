# Public And Participant Surface Design Rules

This file is a surface-specific supplement to [`dashboard-design-system.md`](./dashboard-design-system.md).

## Purpose

The public homepage and the participant room view solve different jobs.

- The public page explains what Harness Lab is and offers a clean entry into room context.
- The participant page answers one practical question: what should I do in this room right now?

They should feel related, but not identical.

## Public Surface Rules

1. The first screen must carry the product.
No oversized dead air above the hero. The workshop identity and participant entry should appear immediately.

2. The access path is primary.
The event-code card is not a footer utility. It is one of the page’s main actions.

3. Supporting principles should be visible early, but compressed.
Users should see the workshop posture without having to scroll through a long manifesto.

4. Navigation should stay light.
Public navigation is orientation, not application chrome.

5. The page should feel calm, but not empty.
Whitespace is useful only when it clarifies hierarchy.

## Participant Surface Rules

1. Show the live phase first.
Participants need the current workshop state before they need background explanation.

2. Make the next action legible.
Current phase, next block, session expiry, and shared notes should be visible without digging.

3. Room data should read like working context, not raw dump output.
Team checkpoint and repo URL need enough structure to scan quickly on a laptop or phone.

4. Session controls should stay secondary.
Leaving room context matters, but it should not dominate the page.

5. Shared notes should be ambient and actionable.
They should feel like lightweight room signals, not like a dense feed.

## Shared Visual Rules

1. Use one calm visual family across all dashboard surfaces.
2. Prefer soft grouping over piles of identical boxes.
3. Use stronger contrast only for decisive actions.
4. Keep the tone lowercase, precise, and quiet.
5. On mobile, every section should collapse into a clear vertical reading order.
