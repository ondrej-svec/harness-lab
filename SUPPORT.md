# Support

Use the shortest path for the type of help you need.

## Usage Questions

Open a GitHub issue if you need help with:

- installing `@harness-lab/cli`
- running the dashboard locally
- understanding the workshop blueprint
- deploying your own instance

When possible, include:

- what you were trying to do
- the command or page you used
- the error message or unexpected behavior
- whether you were in local file mode or hosted Neon mode

## Bug Reports

Open a GitHub issue for reproducible bugs.

Good bug reports include:

- exact steps to reproduce
- expected behavior
- actual behavior
- relevant logs, screenshots, or test output
- environment details such as Node version, OS, and whether you are using Vercel or Neon

## Security and Sensitive Data

Do not post secrets, credentials, participant data, or real workshop logistics in public issues.

If the problem involves private data, auth, or a vulnerability, contact the maintainers privately instead of opening a public issue.

## Contributions

If you already know the fix, open a pull request and follow:

- [docs/contributing.md](docs/contributing.md)
- [CODE_OF_CONDUCT.md](CODE_OF_CONDUCT.md)

## Scope Note

Harness Lab is an open workshop template, not a managed support service. Maintainers may not be able to provide fast turnaround, operational consulting, or custom workshop implementation support for every request.
